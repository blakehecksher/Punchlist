# 2026-03-11 0200 RowHeight

## TL;DR
- What changed: Switched items-grid from CSS grid to flex column; added .item-row wrappers with flex: 1; removed fixed min-height from item cards.
- Why: CSS grid rows don't flex-grow to fill height. Needed flex rows that stretch to fill the page.
- What didn't work: Row heights are still unequal when multiple rooms share a page. The flex tree nests room-blocks between page-content and item-rows, breaking equal distribution.
- Next: Flatten item-rows to page-content level, OR set room-block flex proportional to row count.

## Full notes

### The problem
Flex tree on a page with 406 Hall (3 items = 2 rows) + 407 Family Room (2 items = 1 row):

```
.page-content                   flex column
  .room-block  flex: 1          → 50% of page height
    .items-grid  flex: 1
      .item-row  flex: 1        → 25% of page height (half of 50%)
      .item-row  flex: 1        → 25% of page height
  .room-block  flex: 1          → 50% of page height
    .items-grid  flex: 1
      .item-row  flex: 1        → 50% of page height (all of 50%)
```

407's row is twice as tall as 406's rows. Not what we want.

### Two possible fixes

**Option A — Proportional flex on room-blocks**
Set `flex: <rowCount>` on each room-block. Room-blocks compete at the same level, each getting space proportional to their rows. Simple CSS change. Downside: room headers (which have fixed height) slightly throw off the math — a room with 1 item but a header gets the same space as 1 item row.

**Option B — Flatten item-rows to page-content**
Don't nest item-rows inside room-blocks. Instead, render a flat list at the page-content level: room-header divs interspersed with item-row divs, all as flex siblings. Every item-row is a direct child of page-content and competes equally. Room headers are fixed height (not flex: 1). This is the cleanest solution but requires restructuring the render and potentially the paginate() output.
