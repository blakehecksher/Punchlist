### Storage design
- **localStorage key**: `jbma_punchlist_925park` — stores JSON with `photo: null` everywhere (same `stripPhotos` pattern as before)
- **IndexedDB**: `jbma_punchlist_photos` DB, `photos` object store, keys = item IDs (e.g. `r409_3`, `gn1`)
- On mount: load localStorage → load all IDB photos → merge photos into state
- On photo add: save to IDB immediately; state update propagates to localStorage save (but photo field stripped)
- On photo remove: delete from IDB immediately
