# 2026-03-11 0100 Pagination

## TL;DR
- What changed: Fixed page sizing, row height, and multi-room packing on pages.
- Why: Pages were unconstrained height; rows had fixed min-height instead of stretching; rooms flushed to new pages too aggressively.
- What didn't work: n/a
- Next: UI iteration as directed.

---

## Full notes

### Layout changes
- `.page` now `width: 11in; height: 8.5in` — fixed landscape dimensions on screen and in print
- `.page` uses `display: flex; flex-direction: column`
- New `.page-content` div takes `flex: 1` — stretches to fill remaining height after header/site conditions
- `.room-block` uses `flex: 1; display: flex; flex-direction: column` — rooms share vertical space evenly
- `.items-grid` uses `flex: 1` — fills the room block
- Removed `min-height: 160px` from `.item-card` and `.photo-cell` — height now comes from flex

### Pagination changes
- `ROWS_PAGE_1 = 2` — page 1 gets 2 item rows (site conditions consume ~1 row worth of vertical space)
- `ROWS_OTHER = 3` — pages 2+ get 3 item rows
- Rooms pack together: multiple rooms share a page as long as total item rows ≤ cap
- No more "flush when rowsUsed === ROWS_PER_PAGE - 1" — simpler: just flush when full

### Print CSS
- `@page { size: 11in 8.5in; margin: 0 }` — explicit landscape
- Page padding moved to `.page` rule so it applies in both screen and print
